## PINN-Lab App Redesign — Per-Category View Kits + Animated Dynamics

**Author:** Lead architect, PINN-Lab · **Status:** Proposed · **Repo:** `CAOS_PINNLAB`
**Anchors in code:** `frontend/src/components/CaseExperiment.tsx` (4-tab workbench), `frontend/src/components/FieldView.tsx` (the universal heatmap), `frontend/src/components/LivePanel.tsx`, `frontend/src/lib/contract.ts` (`CaseManifest`, `FieldTrace`), `data-pipeline/pinnlab/cases/base.py` (`CaseSpec`), `data-pipeline/pinnlab/core/{manifest,trace}.py`.

### 0. Problem (grounded in the current code)

`CaseExperiment.tsx:80-89` hard-wires **every** case's Field tab to `<FieldView>` — a viridis heatmap of `trace.fields[outName]` over `manifest.field_axes`. `FieldView.tsx:28` is a single-output, static, 2-D primitive. The audit confirms all 19 cases render identically; 13 are time-evolving but frozen into discrete variant snapshots, 1 is a vector field demultiplexed into scalar tabs (`CaseExperiment.tsx:70-78`), and the dynamical-systems/chaos class is entirely missing.

**The fix is structural, not cosmetic.** Today the *physics category* (`canonical-benchmark`, `mining-mineral-processing`, …) is an organizational bucket with no rendering consequence. We introduce an orthogonal **system-type** classification that *selects the view*. `FieldView` becomes **one kit (`HeatmapKit`) among several**, chosen per-case by a new manifest field `view_kit`. The 6-page product structure and the 4 sub-tabs (Field / Live / Charts / Context) are **untouched**; only what renders *inside* Field and Live changes.

---

### A. Revised system-type taxonomy and the 19-case re-map

We keep the existing 5 *physics* categories on `CaseSpec.category` (they drive Experiments/Benchmark grouping and the case index — do **not** repurpose them). We add a NEW, orthogonal axis: **`system_type`** on `CaseSpec`, and a **`view_kit`** that the manifest derives from it. The mapping `system_type -> default view_kit` lives in one table; `view_kit` can be overridden per case.

| system_type | Default view_kit | Existing cases that re-map here |
|---|---|---|
| `ode-dynamical` | `TrajectoryAnimationKit` | *(none today — new flagship + Lorenz/Duffing/VdW/3-body)* |
| `time-evol-1d` | `TimeEvolutionKit` | `bench-heat1d`, `bench-wave1d`, `bench-burgers1d`, `bench-allencahn`, `env-soil-heat-real`, `poll-soil-barrier`, `poll-tailings-seepage`, `mine-comminution-pbe`, `mine-thickener-settling`, `mine-flotation-kinetics`, `mine-heap-leach-rt` (z-t slice) |
| `time-evol-2d` | `SpatioTemporalKit` | `poll-ocean-transport` (already bakes the `[x,y,t]` cube) |
| `steady-elliptic` | `HeatmapKit` (unchanged) | `bench-poisson2d`, `ctrl-zero-source`, `poll-tailings-seepage` (steady-head variant) |
| `vector-flow` | `VectorFieldKit` | `bench-navier-cavity` (already bakes `u,v,p`) |
| `inverse-assim` | `InverseOverlayKit` | `ind-heat2d-inverse`, `env-soil-heat-real` (recovers α) |
| `eigen-modal` | `ModeShapeKit` | `ind-helmholtz` (resonant-mode variant), `bench-wave1d` (standing-wave mode) |
| `uq-bayesian` | `UQBandKit` | `poll-source-uq-bpinn` (bakes `c`, `c_std`) |
| `operator-surrogate` | `HeatmapKit` + input-fn selector (Phase 4) | `bench-darcy-operator` |

Notes that keep us honest:
- A case can legitimately match **two** kits (e.g. `env-soil-heat-real` is `time-evol-1d` AND inverse). Resolution: the **primary** `view_kit` drives Field; the **Charts** tab hosts the secondary view (the parameter-convergence trace for the inverse aspect). Never cram two heroes into one panel.
- `poll-source-uq-bpinn` is time-evolving *and* UQ — `UQBandKit` composes the `TimeEvolutionKit` scrubber (band animates with the mean), so no conflict.
- `ind-helmholtz` ships **today** as a single forced field (`u[x,y]`). Promoting it to `ModeShapeKit` requires the pipeline to bake a mode family (`pinnlab.modal/v1`). Until that bake exists it stays on `HeatmapKit` — **no overclaim**.

### B. View-kit catalogue

Each kit is a React component `frontend/src/components/kits/<Kit>.tsx` registered in `frontend/src/components/kits/registry.ts` (`Record<string, KitComponent>`). `CaseExperiment.tsx` replaces lines 67-98 with `const Kit = KITS[manifest.view_kit] ?? HeatmapKit; ... <Kit manifest trace active lang/>`. The `KitComponent` prop contract is uniform: `{ manifest, trace, active, lang, onnxEval }`.

All kits honor the interactive-visualization rubric: **hover value-readout**, **marking**, **lib-per-data-type**, **decimation/perf budget**, **controls react live**, **a11y SR-fallback table**.

| Kit | Renders | Key interactions | Lib | Trace/manifest data it needs | Dynamic? |
|---|---|---|---|---|---|
| **HeatmapKit** (= today's `FieldView`, refactored in place) | viridis `u(x,y)` + crosshair readout + colorbar + 2 line-cuts; ADD marching-squares iso-contours + color-scale clip | hover (x,y,u); zoom/pan; contour toggle+levels; live ONNX refine on deep zoom | Canvas2D `putImageData` (existing) + uPlot for the cuts | `fields[out]=[x,y]`, `axes{x,y}` — **already baked** | no |
| **TimeEvolutionKit** | HERO = animated `u(x)` line playing over t (ghost of t=0 + fading trail); SECONDARY = the `[x,t]` carpet as a **seek-bar** with a t-cursor line | play/pause/scrub/loop/speed/step; hover (x,u); click carpet to seek; y-autoscale-lock; front-tracking marker | uPlot (`setData` per frame, 81-pt < 1ms) + Canvas2D carpet | `fields[out]=[space,t]` + `axes` — **already baked** (the trace IS the frame sequence; animation indexes t-columns) | **yes** |

| **UQBandKit** | mean line + filled ±1/2σ band (+ optional spaghetti) animated over t, OR dual synced mean/σ heatmaps; calibration coverage stat shown | hover (mean,σ,band); scrub t; band-width toggle; high-σ shading; spaghetti on/off | uPlot (band+spaghetti) + Canvas2D dual maps | `fields={c,c_std}` + `summary.uq_calibration_2sigma` — **already baked (uq-bpinn)**; spaghetti needs optional `c_members[K,x,t]` | **yes** |
| **InverseOverlayKit** | reconstructed field + sparse OBSERVATION markers (colored by measured value + residual halo) + parity scatter (1:1, holdout RMSE) + parameter-convergence trace (→ true, dashed) | hover field / marker (measured/pred/residual); click-link to parity; layer toggles; convergence scrub | Canvas2D field + SVG markers + uPlot (parity+convergence) | `fields={k,T}` — baked; **NEEDS** the empty `trace.inverse` block populated: `{observations:[…], param_history:{…}, param_true, param_recovered}` | no (optional iter-scrub) |

**Why these libs (budget discipline):** uPlot (~48KB) is the substrate for every line/series/spectrum/band and gives free cursor-sync + zoom; Canvas2D `putImageData` (the existing `FieldView` path) handles every heatmap/linkage/Poincaré frame at 60fps **with zero WebGL contexts**, so many kits coexist within the 8-16 context budget. three.js is the only WebGL dep and is **lazy + toggle-gated**. Plotly is rejected everywhere (~3MB for less control).

### C. New cases + dynamic re-presentation of existing cases

**New `ode-dynamical` cases** (new `cases/dyn_*.py`, each a `CaseSpec` with `system_type="ode-dynamical"`, `view_kit="TrajectoryAnimationKit"`, `validation_anchor="integrator-ref"`):

1. **`dyn-double-pendulum`** — THE FLAGSHIP. `N: t -> (θ₁,θ₂)`; AD gives `ω_i, θ_i''`; residuals on the two closed-form accelerations; **hard IC ansatz** `θ̂ᵢ(t)=θᵢ(0)+t·ωᵢ(0)+t²·Nᵢ(t)` (mirrors `bench-wave1d`'s transform). Training: causal/time-weighted + sequential time-marching + horizon curriculum + L-BFGS polish. **Data-physics hybrid**: sparse anchor points (real tracked-video dataset → `validated-real`, or noisy RK45 → `synthetic-illustrative`). Anchor = adaptive RK45 (`rtol=atol=1e-10`) + symplectic energy check. Variants: `chaotic-120deg` (IC 120°), `mild-30deg`, `twin-IC-δ1e-3` (butterfly panel). Honest headline metric = **leave-time** (where PINN leaves RK45 within tol) and the Lyapunov-horizon line.
2. **`dyn-lorenz`** — `t -> (x,y,z)`; the canonical case proving causal training fixes vanilla-PINN chaos failure. Variants: `classic-σ10ρ28`, `causal-on/off` (shows the methodology). 3-D butterfly via SurfaceKit-style three.js orbit (composed into the linkage panel slot).
3. **`dyn-duffing`** — `(t,γ) -> x` **parametric** net; γ slider in Live walks period-1→2→4→chaos through ONE shared ONNX (exactly the repo's parametric pattern). Stroboscopic Poincaré is the hero; live bifurcation diagram.
4. **`dyn-vanderpol`** — `(t,μ) -> x`; the **non-chaotic** contrast: all ICs spiral onto one limit cycle (PhasePortraitKit basin overlay), isolating *stiffness* from *chaos*. μ slider morphs sinusoidal→relaxation.
5. *(stretch)* **`dyn-three-body`** — planar figure-eight vs chaotic ejection; symplectic/energy-penalty story; deferred to Phase 3 tail.

**Existing cases re-presented dynamically (NO retraining — the trace already holds the frames):**
- `bench-heat1d`, `bench-wave1d`, `bench-burgers1d`, `bench-allencahn`, `env-soil-heat-real`, `poll-soil-barrier`, `poll-tailings-seepage`, `mine-comminution-pbe`, `mine-thickener-settling`, `mine-flotation-kinetics` → **TimeEvolutionKit** (animate the existing `[space,t]` trace by indexing t-columns).
- `poll-ocean-transport` → **SpatioTemporalKit** (the `[x,y,t]` cube already exists).
- `bench-navier-cavity` → **VectorFieldKit** (the `u,v,p` grids already exist; streamlines computed client-side, no new bake).
- `poll-source-uq-bpinn` → **UQBandKit** (mean+std baked).
- `ind-heat2d-inverse` → **InverseOverlayKit** (needs the `inverse` block populated; field works now).
- `ind-helmholtz` → **ModeShapeKit** *after* the modal bake exists; `HeatmapKit` until then.

### D. Phased plan (smallest-impact-first; each phase independently shippable + screenshot-verified + deployed)

**Phase 0 — plumbing (no visual change, ships green).** Add `system_type` + `view_kit` to `CaseSpec` (`base.py`) and `CaseManifest` (`manifest.py` + `contract.ts`), default `view_kit="HeatmapKit"`. Build the kit registry (`kits/registry.ts`), refactor `FieldView` into `kits/HeatmapKit.tsx`, route `CaseExperiment` through `KITS[manifest.view_kit]`. Every case still renders the heatmap — **zero regression**, the seam is now in place.

**Phase 1 — TimeEvolutionKit + SpatioTemporalKit (HUGE payoff, NO training).** Ship the two animation kits, set `view_kit` on the 11 time-1D cases + ocean-transport. 13 cases instantly come alive from already-baked traces. Screenshot-verify play/pause/scrub on every retrofitted case (verify EVERY panel, edge-case sources). **This is the visual win at lowest risk.**

**Phase 2 — TrajectoryAnimationKit + the double-pendulum pipeline.** Add `pinnlab.trajectory/v1` trace flavor (`trace.py` + `contract.ts`), the `integrator-ref` anchor, and `dyn-double-pendulum` (ODE-residual training, hard-IC, causal+time-marching, RK45 anchor, hybrid data). Ship the flagship + Lorenz. Hero: animated linkage + phase portrait + Poincaré + energy-drift + butterfly panel + honest leave-time.

**Phase 3 — VectorFieldKit + UQBandKit + remaining dynamical.** VectorFieldKit (navier-cavity), UQBandKit (uq-bpinn), Duffing + Van der Pol (parametric ODE nets, live γ/μ sweep). PhasePortraitKit falls out of the Trajectory work.

**Phase 4 — ModeShapeKit + InverseOverlayKit + operator surrogate.** Requires new bakes: `pinnlab.modal/v1` (helmholtz mode family) and populating `trace.inverse` (observations + param-history) for heat2d-inverse/soil-heat. Darcy-operator gets an input-function selector. Highest pipeline cost, lowest count → last.

### E. Honest validation + known-limits per kit (surfaced in-UI)

- **TrajectoryAnimationKit (chaos):** anchor = RK45 `1e-10`; show PINN ghost over RK45 with the **divergence marker**; headline = **leave-time**, NOT long-term match. Surface: **Lyapunov horizon** `t_pred ~ (1/λ_max)·ln(1/ε₀)`; the **"PINNs cheat" / no-IC-sensitivity** caveat (soft IC → wrong nearby trajectory; we use hard IC); **energy drift** monitored as a gauge (penalty trades residual vs invariant accuracy — report both).
- **TimeEvolutionKit:** mark known PINN failure regimes — Allen-Cahn stiff front, high-speed convection, inviscid Burgers shock; overlay analytic reference where baked; show l2_relative band per variant (from `active.metrics`).
- **VectorFieldKit:** pressure determined up to a constant (annotate); high-Re/boundary-layer out of vanilla-PINN reach; show Ghia l2 from `summary`.
- **UQBandKit:** show `uq_calibration_2sigma` coverage; never hide high-σ regions; state HMC-vs-VI calibration caveat.
- **InverseOverlayKit:** ill-posedness/identifiability; holdout RMSE distinguished from training sensors; convergence-to-true with the dashed reference.
- **ModeShapeKit:** higher modes harder (spectral bias = more nodes); degenerate-eigenvalue flag; only K baked modes are claimed.
- **HeatmapKit:** the ONE honest still-field class; high-k Helmholtz spectral-bias note.
- Global: `real_or_synthetic` badge stays prominent — only `env-soil-heat-real` (and a real-data double-pendulum if used) are `validated-real`; everything else labeled synthetic/illustrative. **1/19 real today — no overclaim.**
